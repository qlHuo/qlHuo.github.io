初始化项目时遇到的问题

脚手架安装问题

问题描述：

在此项目之前我已经之前安装过了脚手架，但是是@vue/cli4.x版本的，而老师的项目用的是vue-cli2.x。为了避免学习过程中的兼容性等问题，我就准备使用与老师同样版本的vue-cli2.9.6。然后问题就来了：虽然可以安装vue-cli2.9.6，但是使用不了，并且之前的@vue/cli4.2.3也无法完全卸载，然后就很挠头了。。

解决方案：

参考文章

原因:

之前进行过node环境的配置，但是中间有些路径没有填写正确，导致后面卸载不掉新版的@vue/cli脚手架。实际上，其他工具包估计也会出现这个问题，但是，我没有注意过呀。。

原因: npm install -g @vue/cli 后， 我发现在 C:\Users\Administrator\AppData\Roaming\npm\node_modules 目录下多了一个@vue的文件夹。也就是说全局安装的文件都在这里！ 同时，npm uninstall vue-cli -g 也是删除的这里面的文件。

解决：

1.打开环境变量>在用户变量中找到path,这里的的路径必须和你电脑 npm 的全局安装路径对应，原来我的之前路径写错了，所以就默认帮我装在C:\Users\Administrator\AppData\Roaming\npm中！ 现在我决定指定这个路径在D:\nodejs\node_global中（这里的node_global文件夹是我自己新建的）

2.接下来还有一点！系统变量中的NODE_PATH的路径必须和你是 npm 全局安装路径下的 node_modules,所以我这里填写的是D:\nodejs\node_global\node_modules

3.设置 npm 的默认安装路:

$npm config set prefix "D:\nodejs\node_global" //这里的路径必须是上面path对应！

$npm config set cache "D:\nodejs\node_cache" //—这里是我自定义的缓存路径，也配置一下吧。

注意：C:\Users\Administrator\AppData\Roaming\npm，，一般安装时，没修改 node 安装路径，默认的 npm 全局安装路径就在这里

npm run dev 报错

webpack-dev-server错误法则：

前往项目根目录删除node_modules文件夹，然后在项目根目录路径下的终端运行”npm install”等待安装完之后，再次运行“npm run dev”，有些人的是马上就可以了，然而往往还会有人（譬如我）仍然报类似的错误，这个时候你只需要再次重复相同的操作即可，“一次不成再删再安装”！！！

嗯，也是绝了，一次不行就多整几次，这部分参考这篇文章。

多页面应用VS单页面应用

多页面应用

每次页面跳转，后端返回一个新的html文档

优点：

  1. 首屏时间快（页面首个屏幕的内容展现速度）
  2. SEO效果好（搜索引擎可以识别首页中的跳转链接）

缺点：

页面切换慢（每次跳转页面需要发送HTTP请求）

单页面应用

JS 感知路由（URL）变化，动态清除页面内容并将新页面的内容挂到页面上（JS渲染），此时由前端实现路由

优点：

页面切换速度快

缺点：

JS首屏时间慢，SEO差

项目代码初始化

移动端视口标签配置

在 index.html的 meta标签上补充 minimum-scale=1.0,maximum-scale=1.0,user-scalable=no，设置屏幕的最小缩放比和最大缩放比，以及禁止用户随意调整视口的大小。或者直接覆盖对应的meta标签。更多移动端布局参考这篇文章

<meta name="viewport" content="width=device-width,user-scalable=no,initial-scale=1.0,maximum-scale=1.0,minimum-scale=1.0" />

引入初始化样式

作用：保证项目在不同设备之间显示效果一致。放到assets目录下的styles目录中，并在main.js中引用即可

reset.css

reset.css的主要作用是初始化项目样式，保证不同设备之间显示效果一致。

border.css

border.css的主要作用是：解决某些设备多倍屏1像素边框显示问题。

解决移动端300ms延迟问题

在项目文件中安装fastclick

npm install fastclick --save

在main.js中引入发射台Click，并进行配置如下：

// 引入fastClick文件
import fastClick from 'fastclick'

// 在项目中使用fastclick
fastClick.attach(document.body)
