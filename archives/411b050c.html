路由配置

基本配置

首先需要新建city文件夹，并且新建 City.vue 主页面及其子组件等。

然后需要在 router/index.js 文件夹下配置路由规则，确定城市详情页面的跳转位置，首先需要引入相关的组件，然后配置 路由规则如下：

// 引入组件
import City from '@pages/city/City'
// 为该组件配置路由规则
{
    path: '/city',
    name: 'City',
    component: City
}

router-link的使用

router-link 标签会被渲染为 a 标签。那么如何进行页面间的跳转呢？

在需要跳转DOM元素外使用 router-link 标签包裹，并添加 to 属性，to属性值为 要跳转到的路径。

router-link 有默认的颜色，可以添加自定义的颜色将默认颜色覆盖。

BetterScoll的使用

BetterScroll 是一款重点解决移动端（已支持 PC）各种滚动场景需求的插件。它的核心是借鉴的 iscroll 的实现，它的 API 设计基本兼容 iscroll，在 iscroll 的基础上又扩展了一些 feature 以及做了一些性能优化。

BetterScroll 是使用纯 JavaScript 实现的，这意味着它是无依赖的。仓库地址 中文文档

安装

npm install better-scroll -S # install 1.x
npm install better-scroll@next -S # install 2.x，该版本带有所有插件的能力。

html结构要求

<div class="wrapper">
  <ul class="content">
    <li>...</li>
    <li>...</li>
    ...
  </ul>
  <!-- 这里可以放一些其它的 DOM，但不会影响滚动 -->
</div>

上面的代码中 BetterScroll 是作用在外层 wrapper 容器上的，滚动的部分是 content 元素。这里要注意的是，BetterScroll 只处理容器（wrapper）的第一个子元素（content）的滚动，其它的元素都会被忽略。

引入并初始化

import BScroll from '@better-scroll/core'
let wrapper = document.querySelector('.wrapper')
let scroll = new BScroll(wrapper)

在本项目中，具体实现如下：

// 引入这个组件
import BScroll from 'better-scroll'
// 在页面挂载完成之后，初始化BScroll，通过this.$refs.warpper获取页面的DOM元素
mounted () {
  this.scroll = new BScroll(this.$refs.wrapper, {click: true})
},

使用axios获取城市数据并渲染到页面

在City.vue中通过 axios 获取数据默认返回的是一个Promise对象，我们可以使用async和await简化数据的操作，使数据显示更加清楚。

// mounted 钩子函数是在页面挂载之后执行，当页面挂载之后立即获取city数据。
mounted () {
  this.getCityInfo()
},
methods: {
  async getCityInfo () {
    // axios 请求返回的是一个promise对象
    const { data: res } = await axios.get('/api/city.json')
    // console.log(res)
       if (res.ret && res.data) {
          this.cities = res.data.cities
          this.hotCities = res.data.hotCities
       }
  }
}

父子间传值

我们已经在父组件 City.vue 中获取到了城市数据，然后可以通过父组件向子组件传值的形式使子组件使用到城市的数据。

父组件通过属性绑定的形式向子组件传值，子组件通过 props 接收父组件传递过来的值。

// 父组件使用属性绑定的形式向子组件传值



// 子组件使用 props 接收，然后就可以在页面中使用这些数据了。
props: ['cities', 'hotCities']
或者：
props: {
    cities: Object,
    hotCities: Array
}

列表性能优化

函数节流：

当鼠标在字幕表上来回移动的时候，这个时候 touchMove 执行的频率是非常高的，通过函数节流限制函数执行的频率。通过设置定时器，延迟8ms执行一次可以有效的提高性能。

handleTouchMove (e) {
      if (this.touchStatus) {
        // const startY = this.$refs['A'][0].offsetTop
        // console.log(startY)
        // 函数节流，提高性能
        if (this.timer) {
          clearTimeout(this.timer)
        }
        this.timer = setTimeout(() => {
          const touchY = e.touches[0].clientY - 79
          const index = Math.floor((touchY - this.startY) / 20)
          // console.log(index)
          // console.log(touchY)
          if (index >= 0 && index <= this.letters.length) {
            this.$emit('letterChange', this.letters[index])
          }
        }, 8)
      }
},

搜索逻辑实现

  1. 在 Search.vue 组件的data 中定义 keyword 并通过 v-model 双向绑定到 搜索框中。
  2. 在父组件 City.vue 中向子组件传递 城市列表数据 cities ，在子组件 Search.vue 中使用 props 属性接收。
  3. 在 Search.vue 组件 data 中定义 list 数组，用来存储搜索匹配的 城市列表数据。
// 使用侦听器 监听 keyword 的变化。
watch: {
    keyword () {
      if (this.timer) {
        clearTimeout(this.timer)
      }
      if (!this.keyword) {
        this.list = []
        return
      }
      this.timer = setTimeout(() => {
        const result = []
        // console.log(this.cities)
        for (let i in this.cities) {
          // console.log(i)
          this.cities[i].forEach(value => {
            if (value.spell.indexOf(this.keyword) > -1 || value.name.indexOf(this.keyword) > -1) {
              result.push(value)
            }
          })
        }
        this.list = result
      }, 100)
    }
  }

vuex实现数据共享

State 用于存放共用数据，组件改变数据必须调用Actions，作一些异步处理或批量的通过操作，然后Actions调用Mutations，只有通过改变Mutations才能改变State的值。有时可以跳过Actions，直接调用Mutations。

安装

npm install vuex --save

基本使用

创建一个store文件夹，并且新建一个index.js。在这个js文件中编写vuex代码。

import Vue from 'vue'
import Vuex from 'vuex'

Vue.use(Vuex)
export default new Vuex.Store({

})

在项目入口文件 main.js 中引入

import store from './store'

new Vue({
    el: '#app',
    router,
    store,
    ...
})

localStorage本地存储

localStorage是HTML5 新提供的API，能提供类似cookie功能，做到本地存储，且比cookie更简单化。

注意： 使用localStorage时最好使用 try{}catch(e) {}包裹，因为在某些浏览器中用户关闭本地存储或者使用了隐身功能就会无法运行代码。

定义Vuex的时候，可以吧statte，actions，mutations等参数的内容拆分出来。

方法：

  1. 把state，actions，mutations的内容分别放在单独的js文件中
  2. 通过import引用到index.js中，并在index.js中定义vuex的选项

优化网页性能

keep-alive 会把包裹的页面缓存起来，可以避免多次请求重复的数据。

<keep-alive>
    <router-view></router-view>
</keep-alive>

当路由的内容贾在国一次，就将其存放在内存中，下一次进入不需要重新渲染执行钩子函数，只需要在内存中取，避免多次请求重复的数据。

当引入keep-alive 的时候：

  1. 页面第一次进入，钩子的触发顺序是 created-> mounted -> activated
  2. 再次进入（前进或者后退），只触发activated。
  3. 每次退出是触发 deactivated
