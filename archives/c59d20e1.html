stylus在vue项目中的使用

第一次使用stylus这个css预处理器，也不是很难和less差不多，主要的区别就是代码的书写规则不太一样（废话），可能还有其他的特性，学到的时候再说吧，现在就简单的记录，本次项目实战中的基本使用和一些问题。

安装

终端中运行以下命令。注意：要重新启动编译项目

npm install stylus -save
// stylus 语法的解析器，webpack中的内容
npm install stylus-loader -save

基本用法

在组建中使用stylus编写样式，下面的就是基本的stylus语法，使用缩进表示层级。对于vsCode中stylus配置可以参考这篇文章

  • lang表示使用哪种css预处理器
  • scoped 声明样式仅在该组件中有效
<style lang="stylus" scoped>
    .header
      line-height: 0.86rem
      display: flex
      background-color: $bgColor
      color: #fff
      .header-left
        width: 0.64rem
        float: left
        div
          text-align: center
          font-size: 0.36rem
</style>

使用rem

初始化样式中设置页面根元素的字体大小为 50px，所以在本项目中：1rem = 50px。

本项目中iconfont的使用方法

  1. 官网：http://www.iconfont.cn/选择需要的icon
  2. 需要引入下载后的这四种字体文件.eot.svg.ttf.woff及.css样式
  3. 在.css文件里修改ur1路径加入./iconfont/注意base64不用加
  4. 在main.js中加入 import 'styles/iconfont.css'
  5. 在Header.vue里面各个需要的div或者span标签中加入 class="iconfon t"，中间写各个图标的复制码即可完成

其他关于字体图标的使用，查看这篇文章

代码优化相关

添加全局颜色

添加全局颜色，增强主题样式的可维护性：

  1. 在 styles 文件夹中增加 varibles.styl 文件里面写 $bgColor=#00bcd4 ，之后在 Header.vue 中使用： background：$bgColor。
  2. 在 Header.vue 中的 style下加入 @import'~@/assets/styles/varible s.styl 即可

注意：

  1. .import 在 CSS 中前必须加 @。JS中不用加。
  2. ~@ 在引入CSS文件中表示 src 目录。
  3. @ 在引入 js库的时候表示 src 目录。

自定义路径别名

在 build/webpack.base.conf.js 文件里，找到 reslove

 resolve: {
    extensions: ['.js', '.vue', '.json'],
    alias: {
      'vue$': 'vue/dist/vue.esm.js',
      '@': resolve('src'),
      'styles': resolve('src/assets/styles'),
    }

@就是这么定义的，styles是我们自己为 src/assets/styles 路径起的一个别名，后续就可以直接使用 styles 替代 src/assets/styles。优化了代码。

注意修改路径代码后，会报错，重启项目就好了

仓库代码分支

在实际的项目开发中，我们在实现不同的功能之前需要创建不同的分支，这样不仅有利于协同开发，还有利于保障项目结构清晰和完整。当局部分支出现错误的时候，不会影响到整个项目。

  1. 新建分支： 在github的仓库里面，找到Branch，新建一个分支名为 index-swiper。创建完成之后，线上的github仓库就会多一个 index-swiper 分支。

  2. 拉取分支： 新建分支完成之后，本地仓库并没有 index-swiper 分支，使用以下命令拉取分支，然后本地就有了 index-swiper

    git pull

  3. 切换分支并编写相应的代码：

    // 查看当前所处分支
git branch
// 切换分支 
git checkout index-swiper

  4. 功能完成之后，在原分支上提交本地代码到线上仓库，然后切换到主分支主动合并其他分支：

    // 提交当前分支到远程仓库
git add .
git commit -m '提交代码的描述'
git push   // 提交到了远程仓库中的index-swiper分支

// 切换到主分支
git checkout master 
// 主分支合并刚才的index-swiper分支
git merge index-swiper
// 推送到线上主分支
git push

我在最后合并分支的时候，在git checkout master之后，发现刚刚编写的代码都没了，心里凉半截。。。后来才发现，是因为还没合并，而刚才的代码是在 index-swiper 这个分支上写的，所以，合并之前master分支当然没有刚刚编写的代码了。。

swiper插件的使用

安装： 跟随老师的版本

npm install vue-awesome-swiper@2.6.7 --save

vue-awesome-swiper地址，当前版本为4.11，有一定的差异，使用的时候可以查阅它提供的API或者百度。。

swiper相关的功能

在data中返回一个 options 对象，用来配置swiper提供的内置方法。

 data () {
    return {
      swiperOptions: { // swiper 标签中的 options 的属性值
        pagination: '.swiper-pagination',  // 显示默认的小圆点
        loop: true,  // 可以连贯的滑动
        autoplay: 2000  // 设置自动播放 2s
  },

更改小圆点的样式：

首先可以通过审查元素的方式，找到小圆点对应的 样式类。

然后使用 >>> 标签越过 scoped 标签的限制。应该也可以新建一个global.css，里面专门放置需要修改原插件中样式的内容。

.warpper >>> .swiper-pagination-bullet-active
    background-color: #fff
解决页面加载过程中的抖动

swiper 外层加一个 div 包裹，并为 div 设置样式如下。

.warpper
    overflow: hidden
    width: 100%
    height: 0
    padding-bottom: 30.4%

padding-bottom 的值是图片的高和宽的比例。

icons图标导航区域

导航分页可滑动效果

<template>
    <div class="icons">
      <swiper :options='swiperOptions' style="touch-action: none;">
        <swiper-slide v-for="(page,index) in pages" :key="index">
          <div class="icon" v-for="item in page" :key="item.id">
            <img class="icon-img" :src="item.imgUrl">
            <p>{{item.desc}}</p>
          </div>
        </swiper-slide>
      </swiper>
    </div>
</template>

<script>

export default {
  name: 'HomeIcons',
  data () {
    return {
      swiperOptions: {
        autoplay: false
      }
    }
  },
  props: ['iconsList'],
  computed: {
    // icons 分页
    pages () {
      const pages = []
      this.iconsList.forEach((item, index) => {
        const page = Math.floor(index / 8)
        if (!pages[page]) {
          pages[page] = []
        }
        pages[page].push(item)
      })
      return pages
    }
  }
}
</script>

内容超出显示省略号

这个属于css的内容吧，很久没用了，记录一下。

overflow: hidden;
white-space: nowrap;
text-overflow: ellipsis;

可以把这种常用的代码封装到.styl文件中，以后使用可以直接调用。

ellipsis()
    overflow: hidden
    white-space: nowrap
    text-overflow: ellipsis

在样式中引用该文件，然后在对应的位置直接引用该方法：

@import '~style/ellipsis.styl'

.item-content
  ellipsis()

如果文字溢出不显示省略号，那么可以给 父元素 添加 min-width: 0

使用axios发送ajax请求

基本使用

  1. 安装axios

    npm install axios --save

  2. 引入axios

    import axios from 'axios'

  3. 使用生命周期函数 mounted 获取 ajax 数据。mounted函数是在页面挂载完成之后执行。

  4. 请求url，此处数据为模拟后台数据。

    axios.get('/api/index.json')

  5. axios 返回的是 Promise 对象，可以使用 then 获取数据

  6. 模拟后台数据

    在static文件夹下创建 mock 文件夹，添加 index.json 文件

  7. 我们不希望提交 static下面的文件。

    需要在.gitignore文件中添加不需要提交的文件路径（static/mock）

【转发机制-代理】

需求：当请求地址为/api开头时，并没有这个路径，那么我们可以把该请求发送到本地的 /static/mock目录。

目的：模拟真实路径防止上线后再修改请求路径而出现问题

方法：在 config/index.js 里面的 dev -> proxyTable中 设置如下：

proxyTable: {
      '/api': {
        target: 'http://localhost:8080',
        pathRewrite: {
          '^/api': '/static/mock'
        }
      }
    },
